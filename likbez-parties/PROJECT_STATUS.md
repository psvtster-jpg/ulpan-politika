## Что выяснили про публикацию

Текущее решение после проверки вариантов: публиковать через **GitHub + Vercel**.

Что уже выяснили про Vercel:

- Vercel plugin установлен локально в Codex;
- плагин содержит skills, включая `vercel-cli`, `deployments-cicd`, `vercel-api`, `env-vars`, `observability`, `nextjs`;
- отдельной callable-кнопки или MCP tool вида “создать деплой” плагин не дает;
- инструкции Vercel plugin ведут к работе через Vercel CLI или через Git-интеграцию;
- для долгосрочного проекта лучше GitHub-интеграция Vercel, а не ручной CLI-деплой.

Следующий рабочий этап:

1. Создать GitHub-репозиторий для сайта.
2. Положить туда содержимое `likbez-parties`.
3. Подключить репозиторий в Vercel через **Add New -> Project**.
   - Если в GitHub-репозитории лежит весь рабочий каталог, в Vercel указать **Root Directory**: `likbez-parties`.
   - Build Command оставить пустым или `None`.
   - Output Directory оставить пустым или `.`.
4. Получить постоянную ссылку вида `https://...vercel.app`.
5. После появления ссылки обновить `index.html`, `robots.txt`, `sitemap.xml` и Open Graph URL.

Примечание: если нужно срочно получить временную ссылку до GitHub, можно вернуться к Vercel CLI preview-деплою.

Проверяли Netlify Drop:

- страница Netlify Drop открывается: `https://app.netlify.com/drop`;
- Netlify пишет, что можно загрузить проект без аккаунта;
- прямой API-запрос к `api.netlify.com/api/v1/drop/token` вернул `404`;
- запрос к `app.netlify.com/api/v1/drop/token` вернул `400 Bad request, missing form`;
- в браузерном API Codex не оказалось метода `setInputFiles`, поэтому автоматически выбрать ZIP-файл через file picker не получилось;
- значит для Netlify Drop нужен ручной выбор файла в браузере.

Проверяли Catbox:

- загрузка HTML-файла прошла;
- но ссылка показывает HTML как текст, а не как страницу;
- поэтому Catbox не подходит как нормальная публичная ссылка на сайт.

Проверяли 0x0.st:

- загрузки отключены из-за спама;
- не подходит.

## Как опубликовать сейчас

Основной рабочий путь:

1. Подготовить GitHub-репозиторий, например `ulpan-politika-site`.
2. Загрузить туда текущий сайт из папки `likbez-parties`.
3. В Vercel открыть **Add New -> Project**.
4. Выбрать GitHub-репозиторий.
5. Настроить проект как статический сайт.
6. Дождаться деплоя и получить ссылку вида:

```text
https://...vercel.app
```

7. Эту ссылку можно пересылать в Telegram/Facebook.

## Что нужно сделать после публикации

Когда появится публичная ссылка Vercel:

1. Открыть опубликованную страницу и проверить:
   - первый экран;
   - фильтры партий;
   - блок “Главное за 30 секунд”;
   - ссылки Telegram/Facebook.
2. Заменить `https://example.com/likbez/israel-parties` в:
   - `index.html`;
   - `robots.txt`;
   - `sitemap.xml`.
3. Заменить `og:image`, если нужно использовать постоянный домен/хостинг.
4. Проверить, что Vercel автоматически пересобирает сайт после изменений в GitHub.
5. Добавить домен, если будет куплен или выбран постоянный домен.
